## 1. 題目與主題關聯性

**題目**：Roguelike 風格**可玩遊戲原型** — 玩家在程序化生成的地城中移動，以抵達樓梯為勝利目標。

**與「遊戲」關聯**：具完整遊玩循環（生成關卡 → 操作 → 勝負判定），屬電腦遊戲之最小可行遊玩體驗；核心玩法為探索與空間導航。

**MVP 邊界（5/12 前可展示）**

- 已有：隨機種子、地城生成、碰撞、勝利判定、CLI 與 Python Host 雙前端。
- 可選加值（5/12～6/12）：敵怪與戰鬥、物品、音效、圖形化介面（如 pygame）、存檔。

## 2. 預期系統架構

```
Python Host (host/app.py)
    │ ctypes 呼叫
    ▼
libroguecore.so  ← C：rng、dungeon、game 狀態機
```

- **C 層職責**：可重現 RNG、房間+走廊演算法、地磚緩衝區（row-major `uint8`）、玩家座標、移動合法性、`rc_game_done`。
- **Host 職責**：載入 `.so`、將地磚對應為 ASCII、讀取使用者輸入、呼叫 `rc_game_move`。
- **公開介面**：見 `include/roguecore.h`（與書面報告「模組邊界」一致）。
